# Verb Paradigm Boundary

Use this file when a verb-form question cannot be answered safely from productive morphology alone. Georgian verb logic is powerful, but many concrete forms depend on the accepted paradigm of a particular verb.

This file is not a verb dictionary. It defines when to stop rule-only reasoning and verify the concrete paradigm.

## Core Boundary

Use productive rules for structure. Use paradigm verification for concrete stems.

The rule route can show what kind of form is needed. It cannot always determine which stem, theme sign, or participial form a specific verb uses in strict standard Georgian.

## Rule-Decidable Verb Questions

Usually decide locally when the issue is:

- person marker placement
- `თქვენ` plural/polite agreement
- participant role: actor, object, beneficiary, indirect object
- active/passive/benefactive/causative relation when the verb pattern is clear
- third-series inversion as a structural warning
- `არ` vs `ნუ` prohibition type
- whether a form is suspicious because it was created by surface analogy

These questions still require careful analysis, but they are governed by productive grammar.

## Paradigm-Required Verb Questions

Verify externally or mark uncertainty when the answer depends on:

- which theme sign the concrete verb uses: `-ავ`, `-ამ`, `-ებ`, `-ობ`, or another pattern
- which stem appears in aorist, perfect, optative, or participle
- whether a present form made by analogy is strict standard Georgian
- whether an `-ო`-final derived verb takes `-ო-` or `-ოვ-` in the target screeve
- whether an evidential/perfect form is `-ია`, `-ავს`, `-ამს`, or another accepted pattern
- whether a participle or result-state adjective is standard, colloquial, archaic, or professional/register-bound
- whether two common-looking forms differ by normative status rather than grammar logic

Do not hide this boundary. If the local references do not settle the paradigm, say that the concrete paradigm must be checked.

## High-Risk Verb Classes

Treat these as high-risk before answering:

- verbs whose present and perfect/aorist stems differ
- verbs with `-ავ` vs `-ამ` alternation questions
- verbs where colloquial forms compete with strict standard forms
- verbs with object-sensitive roots such as carry/lead/take/bring distinctions
- `ი-`, `ე-`, `უ-` version or passive/benefactive contrasts
- causative forms with `-ინ-` or `-ევინ-`
- denominal verbs and `-ო`-final bases
- participles and verbal nouns that look regular but depend on a fixed stem

Add a new class only when it generalizes. Do not add one-off trap words.

## Verification Protocol

When a paradigm check is needed:

1. Identify the lemma or likely dictionary form.
2. Identify the target function: present, future, aorist, optative, perfect/evidential, participle, verbal noun, imperative, or prohibition.
3. Split the candidate forms into markers and stems.
4. Check whether the local rule files decide the class.
5. If not, consult a normative grammar, dictionary, or reliable conjugation/paradigm source.
6. If no source is available, answer with uncertainty or rewrite around the risky form in drafting mode.

For explicit correctness tests, do not invent certainty. For normal writing, prefer a safer construction.

## Answering Behavior

For short form questions:

- If rule-decidable, give the correct form briefly.
- If paradigm-dependent and verified, give the correct form plus the short reason.
- If paradigm-dependent and unverified, say it is a concrete paradigm question and avoid a hard answer.

For multiple-choice tasks, analyze every option before selecting the answer. A form that sounds common can still be nonstandard for strict test norms.
