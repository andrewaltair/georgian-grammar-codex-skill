# Contributing

Thank you for helping improve Georgian Grammar Skill for Codex.

## What Kind Of Contributions Are Useful

- grammar-rule corrections
- punctuation-rule improvements
- clearer wording in existing rules
- better routing between `SKILL.md` and `references/`
- fixes for incorrect or ambiguous guidance
- narrow additions that improve real Georgian output quality

## Before You Change Anything

1. Read `georgian-grammar/SKILL.md`.
2. Keep the skill rule-first.
3. Preserve the current structure:
   - `SKILL.md` stays the short router
   - deep material goes into `references/`
4. Do not turn the skill into:
   - an answer key
   - a trap-word list
   - a memorized test database

## Good Change Shape

Prefer small, focused pull requests.

Good examples:

- fix one wrong rule
- clarify one ambiguity
- add one narrowly scoped reference section
- improve one routing decision

Avoid unrelated refactors in the same pull request.

## Pull Request Checklist

When opening a pull request, explain:

1. what was wrong
2. what you changed
3. why the new rule or wording is better
4. which files changed
5. one or two short Georgian examples, if they help explain the change

If the change is based on a source, mention the source in the pull request description. Do not paste large copyrighted excerpts.

## Style Rules For Contributions

- Use standard literary Georgian as the target norm
- Prefer rule explanations over example collections
- Keep examples short
- Keep `SKILL.md` lean
- Put detail in `references/` when possible
- Preserve existing file names and structure unless there is a strong reason to change them

## How To Contribute

1. Fork the repository
2. Create a branch
3. Make a focused change
4. Open a pull request
5. Describe the change clearly

## Pull Request Title Examples

- `Clarify relative pronoun agreement`
- `Fix punctuation rule for generalizing word plus list`
- `Improve adverb routing guidance`

## Maintainer Review Standard

Pull requests are easiest to merge when they are:

- narrow
- rule-backed
- clearly explained
- consistent with the current skill architecture
